title: "블로그 만들기가 하루 종일 걸릴 줄이야"
categories: blog update
--- 

개발일지를 공개용으로는 적지 않고 노션에 개인적으로 그날 그날 한 것 중 나중에 보고 참고할 만한 것들을 적어놓고는 있었다.
블로그로 개발일지 공유한다는 얘기는 처음에 들었지만, 
쓰는 데 드는 공수 + 나만 알아보는 게 아니라 남들이 읽을 수 있도록 다듬는 데 느는 공수를 생각했을 때 
전자도 꾸준히 안하게 되는데 후자까지 해야한다면 지레 질려서 포기해버릴 수 있다고 생각했기 때문이다.<br><br>
의외로 혼자 보는 용도의 개발일지(what I learned today 느낌)가 생각보다 흡족하게 쌓이고 있었지만,
그에 반해 한달이 되어가도록 나의 프로젝트는 진전이 없었다.(즉 실질적인 진전은 없이 필요없는 삽질이 많았다는 의미)
공개를 통해 도움도 구하고 스스로 채우다 가는 만족감을 위한 스터디는 지양하기 위해서, 
그리고 결정적으로는 github.io가 붙은 블로그는 어떻게 만드는 건지 궁금해서 구글링을 통해 개설했다.
무슨 차이인지 몰라도 도메인주소에 'naver'가 붙은 것보다는 'github.io'가 붙은 개발 블로그의 개발 정보에 더 신뢰가 가는 느낌적 느낌 때문이라는 것.<br><br>
결과적으로는 원하던 대로 블로그 하나 만드는데 cli를 쳐다보며 에러를 여러번 마주치고, 
스택오버플로와 깃헙 디스커션을 뒤져가며 디버깅을 해서 예시 페이지를 만들었다.
오늘의 개발일지는 오늘 블로그 만들면서 마주친 에러들로 채웠다. 
네가 원하던 '개발자 느낌나는 도메인'을 붙이기 위해 개발할 때 많이 하는 것들을 하고 나니 이제 행복하니..?
어쨌든 하루 안에는 돼서 다행이다.
아무리 github.io가 붙었어도 블로그 개설에 이틀 넘어가면 현타가 올 것 같기 때문이다.<br><br>
이제 글을.. 글을 채워야 하는데 역시 공개할 만한 글들은 아닌 것 같네.
근데 어차피 '~~를 위한 ~~tip!'하고 누군가 궁금해 할 만한 내용을 콕 찝어서 깔끔하게 정리한 글이 아니면
개발일지도 결국 일기나 다름없어서 - 심지어 재미도 없을 - 혼자 본다고 생각해도 되겠다. <br><br>

한 줄 요약: 주소창에 github.io 하나 넣어보겠다고 애썼는데 아직 사용법이 낯설다.<br><br>
**이건 그냥 마크다운 잘 작동하나 보려고 넣는 거**
```
# python
def manse(name):
  print(name + " Manse~~!")

manse("sohee")
```
